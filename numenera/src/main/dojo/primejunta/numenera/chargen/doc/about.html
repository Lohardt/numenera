About the Character Creation Utility

The Character Creation Utility is for creating characters for Monte Cook Games' Numenera pen-and-paper role-playing game. You can create characters, advance them, and produce a printable character record for them.

Isn't there an official app for this? If so, why another one?

There certainly is. I made this mostly because it was fun, and coding up the game mechanics was a good way to learn them. However, I also wanted something that produced printable character sheets, since I don't like people messing with their personal electronics while playing.

How do I use it?

Click around. You'll figure it out.

What is it made of?

It is an offline-capable all-Javascript/HTML/CSS webapp written with the Dojo Toolkit.

Tip: If you're on an iPad and make a link on your desktop for it (the button just to the left of the address field), it will run in full-screen, which is kind of nice.

Do you collect any data?

No, other than usual web server traffic logging. If you link to characters you've shared, that request and its query string does get logged in the web server's HTTP logs.

So where do my saved characters go?

In your browser's local data store. That means they'll only be available on the browser you used to create them. You can share them by copy-pasting links, though.

Will I lose the characters if I empty my history, clear my cache, and remove my cookies?

The details depend on your browser, but yeah, you most likely will.

Any way I can "back up" my characters?

Bookmark the "Share (character name)" link. That won't help if I've updated the app to break data compatibility again. The primary purpose of this app (at this point) is to easily fill out and print a character sheet; that's your real backup.

Anything else about my saved characters?

Don't trust this app. I try to maintain back compatibility for characters, but this version is still unfinished and it may break.

Any gotchas with the printable character sheet?

It seems to work OK on Firefox and Google Chrome, but Firefox prints ugly headers and footers on it. For best results, use Chrome.
